Migration Challenges:
- Package naming conflicts: Both mysocial-core and sui-core have the package name "sui-core" in Cargo.toml
- This makes it difficult to have both packages in the workspace and use them together
- For sui-single-node-benchmark and sui-move-build, we've started the migration by adding mysocial dependencies, 
  but can't fully test the changes due to package name conflicts
- The package name conflict prevents us from running tests in the normal way when both packages are in the workspace

Next steps:
1. Complete migration of partially migrated crates
2. Focus on high-priority non-migrated crates
3. Resolve package naming conflicts using one of these approaches:
   a. Temporarily remove one of the conflicting packages from the workspace during testing
   b. Use feature flags to conditionally include dependencies
   c. Create a test script that temporarily updates Cargo.toml files with the right dependencies for testing
   d. Consider renaming the package in mysocial-core's Cargo.toml to avoid conflicts
4. Consider symlinks in crates/migrated/ for better organization
